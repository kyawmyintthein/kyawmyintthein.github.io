Introduction | နိဒါန်း#

Docker is a powerful containerization platform that has revolutionized how we develop, deploy, and manage applications. ဒီ article မှာ Docker အခြေခံများကို step-by-step သင်ယူကြပါမယ်။

Container technology enables developers to package applications with all their dependencies into lightweight, portable units. ဒီနည်းပညာက developer တွေကို application တွေကို သူတို့ရဲ့ dependency တွေနဲ့အတူ ပေါ့ပါးပြီး သယ်ဆောင်ရလွယ်ကူတဲ့ unit တွေအဖြစ် package လုပ်နိုင်စေပါတယ်။

What is Docker? | Docker ဆိုတာ ဘာလဲ?#

Docker is a containerization platform that allows you to:

  • Package applications with their dependencies
  • Run applications consistently across different environments
  • Scale applications easily
  • Isolate applications from each other

Docker ဆိုတာ containerization platform တစ်ခုဖြစ်ပြီး အောက်ပါတွေကို လုပ်နိုင်ပါတယ်:

  • Application တွေကို သူတို့ရဲ့ dependency တွေနဲ့အတူ package လုပ်ခြင်း
  • မတူညီတဲ့ environment တွေမှာ application တွေကို တသမတ်တည်း run ခြင်း
  • Application တွေကို လွယ်ကူစွာ scale လုပ်ခြင်း
  • Application တွေကို တခုနဲ့တခု isolate လုပ်ခြင်း

Key Concepts | အဓိက အယူအဆများ#

Images vs Containers#

Docker Image: A read-only template used to create containers Container: A running instance of an image

Docker Image: Container တွေ ဖန်တီးဖို့အတွက် သုံးတဲ့ read-only template Container: Image တစ်ခုရဲ့ running instance

1# List images | Image တွေ ကြည့်ခြင်း
2docker images
3
4# List running containers | Running container တွေ ကြည့်ခြင်း  
5docker ps
6
7# List all containers | Container အားလုံး ကြည့်ခြင်း
8docker ps -a

Basic Docker Commands | အခြေခံ Docker commands များ#

1. Running Your First Container | ပထမဆုံး Container run ခြင်း#

1# Run a simple hello-world container
2docker run hello-world
3
4# Run an interactive Ubuntu container | Ubuntu container ကို interactive mode နဲ့ run ခြင်း
5docker run -it ubuntu bash

-it flag သုံးတဲ့အခါမှာ:

  • -i: Interactive mode (stdin ကို open ထားခြင်း)
  • -t: Allocate a pseudo-TTY (terminal ခွဲပေးခြင်း)

2. Working with Images | Image တွေနဲ့ အလုပ်လုပ်ခြင်း#

1# Pull an image from Docker Hub | Docker Hub ကနေ image download ခြင်း
2docker pull nginx
3
4# Build an image from Dockerfile | Dockerfile ကနေ image build ခြင်း
5docker build -t my-app .
6
7# Remove an image | Image ဖျက်ခြင်း
8docker rmi image_name

3. Container Management | Container စီမံခန့်ခွဲခြင်း#

 1# Start a stopped container | ရပ်နေတဲ့ container ကို start ခြင်း
 2docker start container_name
 3
 4# Stop a running container | Running container ကို stop ခြင်း  
 5docker stop container_name
 6
 7# Remove a container | Container ဖျက်ခြင်း
 8docker rm container_name
 9
10# View container logs | Container logs တွေ ကြည့်ခြင်း
11docker logs container_name

Creating a Simple Web Application | ရိုးရှင်းတဲ့ Web Application ဖန်တီးခြင်း#

Let’s create a simple Node.js application and containerize it. Node.js application လေးတစ်ခု လုပ်ပြီး containerize လုပ်ကြည့်ပါမယ်။

Step 1: Create the Application | Application ဖန်တီးခြင်း#

 1// app.js
 2const express = require('express');
 3const app = express();
 4const port = 3000;
 5
 6app.get('/', (req, res) => {
 7  res.json({
 8    message: 'မင်္ဂလာပါ! Hello from Docker container!',
 9    timestamp: new Date().toISOString(),
10    environment: process.env.NODE_ENV || 'development'
11  });
12});
13
14app.get('/health', (req, res) => {
15  res.json({ status: 'OK', message: 'Container က အဆင်ပြေနေပါတယ်' });
16});
17
18app.listen(port, '0.0.0.0', () => {
19  console.log(`App running on port ${port}`);
20  console.log('Server က port ' + port + ' မှာ running နေပါတယ်');
21});

 1// package.json
 2{
 3  "name": "docker-myanmar-app",
 4  "version": "1.0.0",
 5  "description": "Sample app for Docker tutorial",
 6  "main": "app.js",
 7  "scripts": {
 8    "start": "node app.js"
 9  },
10  "dependencies": {
11    "express": "^4.18.2"
12  }
13}

Step 2: Create Dockerfile#

 1# Use official Node.js runtime as base image
 2# Node.js ရဲ့ official runtime ကို base image အဖြစ် သုံးခြင်း
 3FROM node:18-alpine
 4
 5# Set working directory | Working directory သတ်မှတ်ခြင်း
 6WORKDIR /app
 7
 8# Copy package.json and package-lock.json | Package files တွေ copy ခြင်း
 9COPY package*.json ./
10
11# Install dependencies | Dependencies တွေ install ခြင်း
12RUN npm ci --only=production
13
14# Copy application code | Application code တွေ copy ခြင်း
15COPY . .
16
17# Create non-root user for security | Security အတွက် non-root user ဖန်တီးခြင်း
18RUN addgroup -g 1001 -S nodejs
19RUN adduser -S nodejs -u 1001
20USER nodejs
21
22# Expose port | Port ကို expose လုပ်ခြင်း
23EXPOSE 3000
24
25# Define startup command | Startup command သတ်မှတ်ခြင်း
26CMD ["npm", "start"]

Step 3: Build and Run | Build လုပ်ပြီး Run ခြင်း#

 1# Build the image | Image ကို build ခြင်း
 2docker build -t myanmar-node-app .
 3
 4# Run the container | Container ကို run ခြင်း
 5docker run -d -p 3000:3000 --name my-app myanmar-node-app
 6
 7# Check if it's running | Running နေလား စစ်ခြင်း
 8docker ps
 9
10# Test the application | Application ကို test လုပ်ခြင်း
11curl http://localhost:3000

Response က ဒီလို ရမှာ ဖြစ်ပါတယ်:

1{
2  "message": "မင်္ဂလာပါ! Hello from Docker container!",
3  "timestamp": "2024-01-25T14:30:00.000Z",
4  "environment": "development"
5}

Docker Compose | Docker Compose သုံးခြင်း#

For multi-container applications, Docker Compose is very useful. Multi-container application တွေအတွက်မှာ Docker Compose က အရမ်းအသုံးဝင်ပါတယ်။

 1# docker-compose.yml
 2version: '3.8'
 3
 4services:
 5  web:
 6    build: .
 7    ports:
 8      - "3000:3000"
 9    environment:
10      - NODE_ENV=production
11    depends_on:
12      - redis
13    restart: unless-stopped
14
15  redis:
16    image: redis:alpine
17    restart: unless-stopped
18    
19  nginx:
20    image: nginx:alpine
21    ports:
22      - "80:80"
23    volumes:
24      - ./nginx.conf:/etc/nginx/nginx.conf
25    depends_on:
26      - web
27    restart: unless-stopped

1# Start all services | Service အားလုံးကို start ခြင်း
2docker-compose up -d
3
4# View logs | Logs တွေ ကြည့်ခြင်း
5docker-compose logs -f
6
7# Stop all services | Service အားလုံးကို stop ခြင်း  
8docker-compose down

Best Practices | အကောင်းဆုံး လေ့ကျင့်ခန်းများ#

1. Use Multi-stage Builds | Multi-stage build သုံးခြင်း#

 1# Build stage | Build အဆင့်
 2FROM node:18-alpine AS builder
 3WORKDIR /app
 4COPY package*.json ./
 5RUN npm ci
 6COPY . .
 7RUN npm run build
 8
 9# Production stage | Production အဆင့်
10FROM node:18-alpine AS production
11WORKDIR /app
12COPY package*.json ./
13RUN npm ci --only=production
14COPY --from=builder /app/dist ./dist
15CMD ["npm", "start"]

2. Use .dockerignore | .dockerignore သုံးခြင်း#

 1# .dockerignore
 2node_modules
 3npm-debug.log
 4Dockerfile
 5.dockerignore
 6.git
 7.gitignore
 8README.md
 9.env
10coverage
11nyc_output

3. Security Best Practices | လုံခြုံရေး အကောင်းဆုံး လေ့ကျင့်ခန်းများ#

 1# Use specific version tags | တိကျတဲ့ version tag တွေ သုံးခြင်း
 2FROM node:18.19.0-alpine
 3
 4# Don't run as root | Root အဖြစ် မ run ခြင်း
 5RUN addgroup -g 1001 -S nodejs && \
 6    adduser -S nodejs -u 1001
 7USER nodejs
 8
 9# Use COPY instead of ADD | ADD အစား COPY သုံးခြင်း
10COPY package*.json ./
11
12# Minimize layers | Layer တွေ လျော့ခြင်း
13RUN apk add --no-cache \
14    dumb-init \
15    && npm ci --only=production \
16    && npm cache clean --force

Troubleshooting | ပြဿနာ ဖြေရှင်းခြင်း#

Common Issues | အဖြစ်များတဲ့ ပြဿနာများ#

1. Container exits immediately | Container က ချက်ခြင်း exit ဖြစ်ခြင်း

1# Check logs | Logs ကြည့်ခြင်း
2docker logs container_name
3
4# Run in interactive mode for debugging | Debug လုပ်ဖို့ interactive mode နဲ့ run ခြင်း
5docker run -it image_name /bin/sh

2. Port binding issues | Port binding ပြဿနာများ

1# Check what's using the port | Port ကို ဘယ်က သုံးနေလဲ စစ်ခြင်း
2lsof -i :3000
3
4# Use different port | မတူညီတဲ့ port သုံးခြင်း
5docker run -p 3001:3000 image_name

3. Permission issues | Permission ပြဿနာများ

1# Fix file permissions | File permission တွေ ပြင်ခြင်း
2sudo chown -R $USER:$USER .
3
4# Check container user | Container user စစ်ခြင်း
5docker exec container_name whoami

Conclusion | နိဂုံး#

Docker is an essential tool for modern software development and deployment. Docker ဟာ ခေတ်သစ် software development နှင့် deployment အတွက် မရှိမဖြစ် လိုအပ်တဲ့ tool တစ်ခုဖြစ်ပါတယ်။

Key benefits include:

  • Consistency: အတူတူပဲ run မယ်
  • Portability: ဘယ်မှာမဆို သုံးလို့ရမယ်
  • Scalability: လိုအပ်သလို scale လုပ်လို့ရမယ်
  • Isolation: တခုနဲ့တခု သီးခြားစီ run မယ်

Start small, experiment with simple containers, and gradually build more complex applications. နည်းနည်းစ ကနေ စပြီး၊ ရိုးရှင်းတဲ့ container တွေနဲ့ စမ်းကြည့်ပြီး၊ တဖြည်းဖြည်း ပိုရှုပ်ထွေးတဲ့ application တွေ build လုပ်ကြပါ။
